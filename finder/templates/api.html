The Represent API

Represent provides a RESTful JSON API to a wealth of information about Canadian political districts and elected representatives.

The API offers five resources: boundary sets, boundaries, postal codes, representative sets, and representatives.

Basics

The base URL for all requests is http://represent.opennorth.ca

Output formats

All resources output JSON. You may access a user-friendly HTML representation of a JSON response by adding format=apibrowser to the query string. Geospatial resources output GeoJSON. You may instead request KML or Well-Known Text by adding format=kml or format=wkt to the query string.

We support JSONP for client-side cross-domain requests. Just add a callback parameter to the query string.

Filtering resources

You may filter lists of resources using query string parameters. Each resource lists the fields you can filter on. For example, to find representatives named “Rodney,” send a request to /representatives/?first_name=Rodney. You can perform a variety of substring queries by appending __querytype to the parameter name, where querytype is one of iexact, contains, icontains, startswith, istartswith, endswith, iendswith, and isnull. A leading i indicates a case-insensitive match. For example, to find representatives whose last name starts with “M” or “m”, send a request to /representatives/?last_name__istartswith=m.

Bulk downloads

The digital boundary files are available under various licenses. Please consult the boundary sets documentation for information on the boundary files. To download all representatives, send a request to /representatives/?limit=2000.

Rate limits

If you make more than 60 requests per minute, you may receive HTTP 503 Service Unavailable responses. If you expect to exceed this rate limit, please contact represent@opennorth.ca.

API changes

If you use this API, subscribe to our mailing list to stay abreast of any changes.

Attribution

Attribution is not required, but, in the interest of spreading the word and encouraging use of this web service, please consider adding something like District and politician data from <a href="http://represent.opennorth.ca/">Represent</a>.

API wrappers

If you’ve written an API wrapper, please contact represent@opennorth.ca to have it listed here.

Boundary Sets

URL: /boundary-sets/

A boundary set is a particular group of districts, like BC electoral districts or Toronto wards.




Filterable fields: name, domain.



Boundaries



URL: /boundaries/



A boundary is a single district.






The external_id field (which is not necessarily present on every resource) is an ID number used for this district in the source data.



The metadata field contains all attributes from the source data. This is unmodified and may be out-of-date or erroneous.



Filterable fields: name, external_id.



Spatial filters



Access all the districts that include a given point at /boundaries/?contains=[latitude],[longitude]. Boundaries that intersect or touch another boundary can be accessed via /boundaries/?intersects=alberta-electoral-districts/highwood or /boundaries/?touches=alberta-electoral-districts/highwood. (What we mean by “intersects” is, in the specific definitions used by PostGIS, “covers or overlaps.”)






Spatial data



To access spatial data, follow the links provided in the shape_url, simple_shape_url, and centroid_url keys. The simple_shape is the district’s shape, simplified to a tolerance of 0.002; this’ll usually look fine on maps and be quicker to load.



Spatial formats



Geospatial resources output GeoJSON. You may instead request KML or Well-Known Text by adding format=kml or format=wkt to the query string.



Postal codes



URL: /postcodes/A0A0A0/



A Canadian postal code. URLs should include the code in uppercase with no spaces.



Note: We don't provide a URL for a list of all postal codes.






The boundaries_centroid field is a list of boundaries that contain the postcal code’s center point, or centroid. (That point comes from geocoder.ca data.) Because a centroid is a single point,
whereas a postal code is a shape (or a collection of points), the list of boundaries based on the centroid will in some cases be inaccurate.



When a government has made available a file linking postal codes to boundaries, there will be a boundaries_concordance field, which will be a list of boundaries linked to that postal code according to the government file. Note that sometimes postal codes cross political boundaries, and so sometimes this call will return, for example, multiple provincial ridings for a single postcode.



This resource also provides representatives_centroid and representatives_concordance lists.



Representative sets



URL: /representative-sets/



A representative set is a political body to which people are elected; for example, the House of Commons or Toronto City Council.






Representatives



URL: /representatives/



A representative is an elected official.



Note: We don't provide a URL for each individual representative.






Filterable fields: name, first_name, last_name, district_name, elected_office, party_name.



Spatial filters



Access all the representatives for a given point at /representatives/?point=[latitude],[longitude]. This is a shortcut: you can get the same results by first querying the boundaries resource with a contains filter, and then using the boundaries returned to query the representatives resource with a districts filter.

Contributing



All representative data comes from scrapers. If you’d like to use fields we’re not currently returning for a particular jurisdiction, or if you’d just like to help out, update our scrapers!



Find the scraper via the scraperwiki_url field on the appropriate boundary set resource, make a copy on Scraperwiki, edit your copy till it works, then e-mail represent@opennorth.ca.


If we don’t yet have a scraper for a particular jurisdiction, you might want to use one of our existing ones as an example. Use the list of fields above as a reference.
